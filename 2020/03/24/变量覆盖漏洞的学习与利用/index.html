一个爱学习的小白

0x00 前言

代码审计中常常会遇到变量覆盖漏洞。由于学过没总结过，于是总结一下。

0x01 变量覆盖漏洞简介

1、变量覆盖指的是用我们自定义的参数值去替换程序原有的变量值，一般变量覆盖漏洞需要结合程序的其它功能来实现完整的攻击
2、变量覆盖漏洞大多由函数使用不当导致
3、经常导致变量覆盖漏洞场景有：$$，extract()函数，parse_str()函数，import_request_variables()使用不当，开启了全局变量注册等。
4、单独的变量覆盖漏洞很难有利用价值，需要和其他漏洞结合起来才能产生利用价值。

0×02 变量覆盖漏洞类型

1、全局变量覆盖

主要利用PHP 指令register_global实现变量覆盖。
register_globals的意思是注册为全局变量。
当register_global=On时，传递过来的值会被直接的注册为全局变量直接使用；
当register_global=Off时，需要到特定的数组里去得到它。

也就是说，当为ON时，变量来源可能是页面的表单，Cookie等；当为Off时，不会出现问题。
示例代码1：
test1.php

当register_globals=Off时，接收的时候应该用$_GET['id']来接受传递过来的值；
当register_globals=On时，可直接使用$id来接受值,也可用$_GET['id']来接受传递过来的值。
如果已经对变量$id赋了初始值，比如$id=0，那么即使在url中有/test1.php?id=1，也不会将变量覆盖。id值为0。

3、函数使用不当

1.extract()函数

extract() 函数从数组中将变量导入到当前的符号表。该函数使用数组键名作为变量名，使用数组键值作为变量值。针对数组中的每个元素，将在当前符号表中创建对应的一个变量。

怎么理解呢？
我的理解是：将传过来的参数和参数值，作为数组中的一个值。然后将数组键名(传过来的参数)作为变量，将数组键值(传过来的参数值)作为变量值。

2.parse_str函数

parse_str() 函数把查询字符串解析到变量中，如果没有array 参数，则由该函数设置的变量将覆盖已存在的同名变量。

parse_str( string $str [,array &$arr ] )
可以将字符串解析成多个变量。
其中$str是必须的，代表要解析注册成变量的字符串，形式为“a=1”。
经过parse_str()函数之后会注册变量$a并且赋值为1。

与parse_str()类似的函数还有mb_parse_str()，用法基本一致。

3.import_request_variables()函数

import_request_variables()函数用在没有开启全局变量注册时(即register_global = Off )，把 GET/POST/Cookie 变量导入全局作用域中。简单来说，调用了这个函数就相当于开启了全局变量注册。

bool import_request_variables( string $types [, string $prefix ] )
其中$type代表要注册的变量，当$type为GPC的时候，则会注册GET、POST、Cookie参数为变量。

0x04 变量覆盖漏洞防范

变量覆盖漏洞最常见的漏洞点是：
在做变量注册时以及在赋值给变量的时候没有验证变量是否存在。
解决方案：
1.使用原始变量：

  • 不进行变量注册，直接用原生的$_GET、$_POST等数组变量进行操作。
  • 需要注册个别变量时，可直接在代码中定义，然后再把请求中的值赋值给它。

2.验证变量存在：

  • 如果一定要进行变量注册，可以在注册变量前先判断变量是否存在。
  • 若使用extract()函数可以配置第二个参数为EXTR_SKIP。使用parse_str函数注册变量前需要先自行通过代码判断变量是否存在。
  • 自行申明的变量一定要初始化，防止被覆盖。
